# 埋点需求编写 Skill - 完整指南

## 📋 项目概述

这是一个专门为埋点需求编写设计的 Skill，目的是：

✅ **避免重复阅读** - 不用每次都翻整个埋点字典  
✅ **快速查询** - 秒速找到相关埋点定义  
✅ **规范验证** - 确保新埋点符合规范  
✅ **自动生成** - 根据需求自动生成埋点定义  
✅ **节省 Token** - 通过快速参考减少 Token 消耗

## ✅ 编写检查清单

编写埋点前检查:

- [ ] Model 名称是否在埋点字典中存在？
- [ ] Event 名称是否遵循"动词+名词"规范？
- [ ] Type 是否为 action/exposure/app_live 之一？
- [ ] 参数是否使用蛇形命名法？
- [ ] 是否包含 from 参数（来源追踪）？
- [ ] 参数取值说明是否清晰？
- [ ] 是否参考了字典中的类似埋点？
- [ ] 是否标注了版本号？

## 🆘 常见问题

**Q: 如何处理跨模块的埋点？**  
A: 在 model 中使用"模块1-模块2"格式，或在 from 参数中标注来源。

**Q: 参数太多怎么办？**  
A: 区分必需参数和可选参数，必需参数放在 params 中，可选参数放在 info 中。

**Q: 如何处理嵌套数据？**  
A: 使用 JSON 格式的 info 字段，或拆分为多个参数。

**Q: 新增的埋点如何版本标记？**  
A: 在修订记录中标注"YYYYMM第X期新增"，例如"2503第一期新增"。
