## 📖 核心概念

### Model (模块)
- 埋点所属的功能模块
- 示例: 主页、新闻、视频、社区
- 必须在埋点字典中存在

### Event (事件)
- 具体的用户行为或系统事件
- 命名规范: 动词+名词
- 示例: 点击视频、加载新闻、曝光评论

### Type (事件类型)
- `action`: 用户操作（点击、输入、滑动）
- `exposure`: 内容曝光（展示、加载）
- `app_live`: 应用生命周期（启动、关闭）

### Params (参数)
- 埋点上报的数据字段
- 使用蛇形命名法: `video_id`, `user_name`
- 必须包含 `from` 参数（来源追踪）

## 🆘 常见问题

**Q: 如何处理跨模块的埋点？**
A: 在 model 中使用"模块1-模块2"格式，或在 from 参数中标注来源。

**Q: 参数太多怎么办？**
A: 区分必需参数和可选参数，必需参数放在 params 中，可选参数放在 info 中。

**Q: 如何处理嵌套数据？**
A: 使用 JSON 格式的 info 字段，或拆分为多个参数。

**Q: 新增的埋点如何版本标记？**
A: 在修订记录中标注"YYYYMM第X期新增"，例如"2503第一期新增"。
