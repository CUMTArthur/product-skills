## 埋点字典数据结构

### 标准列定义

每个模块 Sheet 通常包含以下列：

| 列名 | 说明 | 示例 |
|------|------|------|
| **model** | 埋点模型/模块名 | 主页频道、新闻频道、综合内页 |
| **event** | 事件名称 | 点击、曝光、加载 |
| **type** | 事件类型 | action、exposure、app_live |
| **params** | 参数列表 | from、id、name |
| **取值说明** | 参数的具体取值和含义 | from：上级页面 |
| **info** | 自定义上报内容 | 额外的上报字段 |
| **修订记录** | 版本更新说明 | 2503第一期新增 |

   - 参数: from、use_days、ad_type、source

## 埋点编写规范

### 基本格式

```
model: [模块名]
event: [事件名]
type: [事件类型]
params: [参数1, 参数2, ...]
取值说明: [参数说明]
```

### 参数命名规范

- **from**: 来源/上级页面
- **id**: 内容ID
- **name**: 内容名称
- **type**: 类型分类
- **duration**: 时长（毫秒）
- **source**: 数据来源
- **ad_type**: 广告类型

### 事件类型

- **action**: 用户操作事件（点击、输入等）
- **exposure**: 曝光事件（内容展示）
- **app_live**: 应用生命周期事件
