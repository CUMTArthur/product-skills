### 标准参数库

**通用参数**:
- `from`: 来源/上级页面
- `id`: 内容ID
- `name`: 内容名称
- `type`: 类型分类
- `duration`: 时长（毫秒）
- `source`: 数据来源
- `position`: 列表位置

**生命周期参数**:
- `use_days`: 用户使用天数
- `ad_type`: 广告类型
- `free_memory`: 空闲内存
- `used_memory`: 已用内存

## 编写规范

### 命名规范

**Model 名称**:
- 使用中文模块名
- 示例: 主页频道、数据频道、主队频道、综合内页

**Event 名称**:
- 动词+名词结构
- 示例: 点击视频、加载新闻、曝光评论

**Type 类型**:
- `action`: 用户操作（点击、输入、滑动）
- `exposure`: 内容曝光（展示、加载）
- `app_live`: 应用生命周期

### 参数规范

**参数命名**:
- 使用蛇形命名法: `video_id`, `user_name`
- 避免缩写: 用 `video_id` 而不是 `vid`

**参数取值说明**:
- 明确说明参数含义
- 列举枚举值（如有）
- 示例: `type: 1=点赞, 2=评论, 3=分享`

### 版本标记

新增埋点时标注:
```
修订记录: 2503第一期新增 - 用户搜索行为追踪
```

## 常见埋点模式

### 模式 1: 列表项点击

```json
{
  "model": "[模块名]",
  "event": "点击[内容类型]",
  "type": "action",
  "params": ["from", "id", "position", "name"],
  "取值说明": {
```

## 常见问题

**Q: 如何处理跨模块的埋点？**
A: 在 model 中使用"模块1-模块2"格式，或在 from 参数中标注来源。

**Q: 参数太多怎么办？**
A: 区分必需参数和可选参数，必需参数放在 params 中，可选参数放在 info 中。

**Q: 如何处理嵌套数据？**
A: 使用 JSON 格式的 info 字段，或拆分为多个参数。

**Q: 新增的埋点如何版本标记？**
A: 在修订记录中标注"YYYYMM第X期新增"，例如"2503第一期新增"。
